## Session: Skill exploration and web research attempts

### WHAT
- Confirmed `heavy-file-ingestion` skill is already installed and up to date at `~/.claude/skills/heavy-file-ingestion/`
- Explored OB1 repo for a "read URL list → store summaries" recipe — none exists
- Attempted to use `julie:web-research` skill to fetch a Stuff.co.nz article

### WHY
User wanted to install the heavy-file-ingestion skill and then explore web research capabilities against a specific NZ news article.

### HOW
- browser39 confirmed installed at `~/.cargo/bin/browser39`
- Stuff.co.nz is a pure JS SPA — server returns 67KB JS bundle with zero article content in static HTML
- No `__NEXT_DATA__`, no JSON-LD, no OG meta tags — nothing extractable without a JS runtime
- WebFetch and browser39 both return only the page title

### IMPACT
- **Gap identified:** No recipe in OB1 for batch URL → summarize → Open Brain capture. Would be a good community contribution.
- **Limitation confirmed:** browser39 cannot handle JS-rendered SPAs. Need Playwright/Puppeteer for sites like Stuff.co.nz.
